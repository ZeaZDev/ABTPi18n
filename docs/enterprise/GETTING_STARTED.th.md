# เริ่มต้นใช้งาน UltimatePro Advance Enterprise

คู่มือนี้จะช่วยคุณตั้งค่าและเริ่มใช้งาน ABTPro UltimatePro Advance Enterprise edition

## ข้อกำหนดเบื้องต้น

### ความต้องการของระบบ

**ความต้องการขั้นต่ำ:**
- **CPU**: 8 คอร์ (แนะนำ Intel Xeon หรือ AMD EPYC)
- **RAM**: 16 GB
- **พื้นที่จัดเก็บ**: 100 GB SSD
- **เครือข่าย**: การเชื่อมต่อเฉพาะ 100 Mbps
- **OS**: Ubuntu 20.04 LTS หรือใหม่กว่า, CentOS 8, หรือระบบที่เข้ากันได้กับ Docker

**แนะนำสำหรับ Production:**
- **CPU**: 16+ คอร์
- **RAM**: 32 GB หรือมากกว่า
- **พื้นที่จัดเก็บ**: 500 GB NVMe SSD พร้อม RAID 1
- **เครือข่าย**: การเชื่อมต่อเฉพาะ 1 Gbps พร้อมการสำรอง
- **OS**: Ubuntu 22.04 LTS

### การพึ่งพาซอฟต์แวร์

- **Docker**: 24.0+ และ Docker Compose 2.0+
- **Python**: 3.11 หรือใหม่กว่า
- **Node.js**: 18 LTS หรือใหม่กว่า
- **PostgreSQL**: 14 หรือใหม่กว่า (แนะนำ 15+)
- **Redis**: 7.0 หรือใหม่กว่า
- **Git**: 2.30 หรือใหม่กว่า

### บัญชีและคีย์ที่จำเป็น

1. **คีย์ API ของตลาด**: จากตลาดที่รองรับ (Binance, Kraken, ฯลฯ)
2. **ข้อมูลรับรอง Google OAuth**: สำหรับการยืนยันตัวตน
3. **โทเค็นบอท Telegram**: สำหรับการแจ้งเตือน (ทางเลือกแต่แนะนำ)
4. **รหัสผู้ค้า PromptPay**: สำหรับการประมวลผลการชำระเงิน (ลูกค้าไทย)
5. **คีย์ใบอนุญาต Enterprise**: มีให้เมื่อซื้อ

## การติดตั้ง

### ขั้นตอนที่ 1: โคลนที่เก็บ

```bash
# โคลนที่เก็บ
git clone https://github.com/ZeaZDev/ABTPi18n.git
cd ABTPi18n

# Checkout สาขา enterprise (หากแยก)
git checkout enterprise
```

### ขั้นตอนที่ 2: ตั้งค่าตัวแปรสภาพแวดล้อม

```bash
# คัดลอกเทมเพลตสภาพแวดล้อม enterprise
cp .env.enterprise.example .env

# แก้ไขไฟล์สภาพแวดล้อมด้วยการกำหนดค่าของคุณ
nano .env
```

**ตัวแปรสภาพแวดล้อมที่จำเป็น:**

```env
# การกำหนดค่าฐานข้อมูล
DATABASE_URL="postgresql://user:password@localhost:5432/abtpro_enterprise"
REDIS_URL="redis://localhost:6379"

# ใบอนุญาต Enterprise
ENTERPRISE_LICENSE_KEY="your-enterprise-license-key"

# ความปลอดภัย
SECRET_KEY="your-secret-key-min-32-chars"
ENCRYPTION_KEY="your-encryption-key-32-bytes-base64"

# การกำหนดค่า OAuth
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"
OAUTH_REDIRECT_URI="http://localhost:3000/api/auth/callback/google"

# คีย์ API ของตลาด (จะถูกเข้ารหัส)
# สามารถเพิ่มผ่าน UI ได้หลังการติดตั้ง

# การกำหนดค่า Telegram (ทางเลือก)
TELEGRAM_BOT_TOKEN="your-telegram-bot-token"

# การกำหนดค่า PromptPay (ทางเลือก สำหรับการชำระเงินไทย)
PROMPTPAY_MERCHANT_ID="your-promptpay-merchant-id"
PROMPTPAY_WEBHOOK_SECRET="your-webhook-secret"

# การตรวจสอบ (ทางเลือกแต่แนะนำ)
PROMETHEUS_ENABLED=true
GRAFANA_ENABLED=true

# การตั้งค่า Production
NODE_ENV=production
PYTHON_ENV=production
LOG_LEVEL=info
```

### ขั้นตอนที่ 3: รันสคริปต์การติดตั้ง

รุ่น enterprise มีสคริปต์การติดตั้งอัตโนมัติ:

```bash
# ทำให้สคริปต์สามารถเรียกใช้ได้
chmod +x install.sh

# รันการติดตั้ง
./install.sh --enterprise

# หรือสำหรับการปรับใช้ production
./install.sh --enterprise --production
```

สคริปต์การติดตั้งจะ:
1. ตรวจสอบความต้องการของระบบ
2. ติดตั้งการพึ่งพา
3. ตั้งค่าฐานข้อมูล
4. รันการย้ายข้อมูล
5. สร้าง frontend และ backend
6. กำหนดค่าการตรวจสอบ
7. ตรวจสอบใบอนุญาต enterprise

### ขั้นตอนที่ 4: เริ่มต้นฐานข้อมูล

```bash
# รันการย้ายข้อมูลฐานข้อมูล
cd apps/backend
python -m prisma migrate deploy

# (ทางเลือก) ใส่ข้อมูลเริ่มต้น
python scripts/seed_enterprise.py
```

### ขั้นตอนที่ 5: เริ่มบริการ

**โหมดพัฒนา:**
```bash
docker-compose -f docker-compose.enterprise.yml up -d
```

**โหมด Production:**
```bash
docker-compose -f docker-compose.enterprise.prod.yml up -d
```

### ขั้นตอนที่ 6: ตรวจสอบการติดตั้ง

1. **ตรวจสอบสุขภาพบริการ:**
   ```bash
   curl http://localhost:8000/health
   ```

2. **เข้าถึง frontend:**
   - นำทางไปที่: http://localhost:3000/en/dashboard
   - คุณควรเห็นหน้าเข้าสู่ระบบ

3. **ตรวจสอบเอกสาร API:**
   - นำทางไปที่: http://localhost:8000/docs
   - คุณควรเห็น FastAPI Swagger UI

4. **ตรวจสอบการตรวจสอบ (หากเปิดใช้งาน):**
   - Prometheus: http://localhost:9090
   - Grafana: http://localhost:3001 (ข้อมูลรับรองเริ่มต้น: admin/admin)

## การกำหนดค่าเริ่มต้น

### ขั้นตอนที่ 1: การตั้งค่าบัญชีผู้ดูแลระบบ

1. นำทางไปที่ http://localhost:3000/en/login
2. เข้าสู่ระบบด้วย Google (ใช้ข้อมูลรับรอง OAuth ที่กำหนดค่าไว้)
3. ผู้ใช้คนแรกที่เข้าสู่ระบบจะกลายเป็นผู้ดูแลระบบโดยอัตโนมัติ

### ขั้นตอนที่ 2: เพิ่มคีย์ API ของตลาด

1. ไปที่ **การตั้งค่า** → **คีย์ตลาด**
2. คลิก **เพิ่มตลาดใหม่**
3. เลือกตลาดของคุณ (เช่น Binance)
4. ป้อนข้อมูลรับรอง API ของคุณ:
   - คีย์ API
   - API Secret
   - (ทางเลือก) Passphrase สำหรับตลาดบางแห่ง
5. คลิก **บันทึกและเข้ารหัส**

ระบบจะเข้ารหัสคีย์ API ของคุณโดยอัตโนมัติโดยใช้ AES-GCM ก่อนจัดเก็บ

### ขั้นตอนที่ 3: กำหนดค่าการแจ้งเตือน Telegram

1. ไปที่ **การตั้งค่า** → **การแจ้งเตือน**
2. คลิก **เชื่อมต่อ Telegram**
3. ทำตามคำแนะนำเพื่อเชื่อมโยงบัญชี Telegram ของคุณ
4. กำหนดค่าความชอบการแจ้งเตือน:
   - การดำเนินการซื้อขาย
   - การแจ้งเตือนกำไร/ขาดทุน
   - การแจ้งเตือนระบบ
   - การแจ้งเตือนข้อผิดพลาด

### ขั้นตอนที่ 4: ตั้งค่าการจัดการผู้ใช้ (ฟีเจอร์ Enterprise)

1. ไปที่ **ผู้ดูแลระบบ** → **การจัดการผู้ใช้**
2. กำหนดค่าบทบาทและสิทธิ์ของผู้ใช้:
   - **ผู้ดูแลระบบ**: เข้าถึงเต็มรูปแบบ
   - **นักเทรด**: สามารถสร้างและจัดการบอท
   - **ผู้ดู**: เข้าถึงแบบอ่านอย่างเดียว
3. เชิญสมาชิกในทีมผ่านอีเมล

### ขั้นตอนที่ 5: กำหนดค่าการจัดการความเสี่ยง

1. ไปที่ **การตั้งค่า** → **การจัดการความเสี่ยง**
2. ตั้งค่าพารามิเตอร์ความเสี่ยงทั่วโลก:
   - ขีดจำกัดการสูญเสียรายวันสูงสุด
   - เปอร์เซ็นต์การลดลงสูงสุด
   - ขีดจำกัดขนาดตำแหน่ง
   - ข้อกำหนด stop-loss
3. บันทึกการกำหนดค่า

## บอทเทรดตัวแรก

### สร้างกลยุทธ์แรกของคุณ

1. นำทางไปที่ **แดชบอร์ด** → **บอทใหม่**
2. กำหนดค่าการตั้งค่าพื้นฐาน:
   - **ชื่อ**: บอทตัวแรกของฉัน
   - **ตลาด**: เลือกตลาดที่กำหนดค่าไว้
   - **คู่เทรด**: BTC/USDT
   - **กลยุทธ์**: RSI Cross (หรือกลยุทธ์ที่มีอยู่ใดๆ)

3. กำหนดค่าพารามิเตอร์กลยุทธ์:
   ```json
   {
     "rsi_period": 14,
     "oversold": 30,
     "overbought": 70,
     "position_size": 0.01
   }
   ```

4. ตั้งค่าพารามิเตอร์ความเสี่ยง:
   - Stop Loss: 2%
   - Take Profit: 5%
   - ขนาดตำแหน่งสูงสุด: 0.1 BTC

5. เลือกโหมดการดำเนินการ:
   - **Paper Trading**: ทดสอบด้วยเงินทุนจำลอง (แนะนำก่อน)
   - **Live Trading**: เทรดด้วยเงินทุนจริง

6. คลิก **สร้างและเริ่มบอท**

### ตรวจสอบประสิทธิภาพบอท

1. ไปที่ **แดชบอร์ด** → **บอทที่ใช้งานอยู่**
2. คลิกที่บอทของคุณเพื่อดู:
   - ตำแหน่งปัจจุบัน
   - ประวัติการซื้อขาย
   - P&L (กำไร & ขาดทุน)
   - เมตริกประสิทธิภาพ
3. การอัปเดตแบบเรียลไทม์ผ่าน WebSocket
